Project title

Formula 1 information display for 2014 season

Instruction

The website contains 5 major modules:

  • Registration and login module
  • profile module
  • Team, racer, and grand prix information display module
  • Match result display module
  • Fun forum module
In this document, I will talk about 5 modules in details. Let's get started O(∩_∩)O~

Overview video

Registration and login module

The website starts at home page, a visitor can only view dashboard. which shows visitors a driver championship standing list. Any link is disabled. Visitors can only view a plain page without any modification.

Registration:To become a user first needs to register in order to get full report about 2014 season result. If a visitor succeeds in registering, then he becomes a normal user and the page will be forwarded to starting page. At that time, a user is able to view all information provided by the website. Meanwhile, he or she can follow their favorite drivers and teams. Each registered user has a profile.

Login: Once a visitor is registered, his or her information like username, password, and email will be stored in database. So next time. a user can directly use website through entering his or her username and password. Without registration, when a visitor click on left navigation parts such as team, circuit, and forum, the page will be forwarded to login page.

Profile module

Profile. Registered users are classified into two different types.

  • common user
  • administrator
Common user A registered user is allowed to follow his/her favorite driver and team. When a user clicks on follow, it will be added to profile.The profile page is a complete view of your personal information, and what he or she follows on website. User can edit and update their profiles.

Administrator is responsible for user mamagement. So he is accessible to user information including username, password and email. At the same time, admin can edit user information and even delete a user. On the other hand, admin is suppoesd to launch a topic for forum and collect user ideas. Forum part will be introduced later.

Team, racer, and grand prix information display module

I show team, racers and grand prix information with the help ofErgast Developer API. This is a really fantastic web service which provides a historical record of motor racing data for non-commercial purposes.

Team In team display page, I show all 11 teams entering 2014 F1 season and display them in decreasing order according to their final points. Users can see team name, team badge and country each team belonging to. On a click of team's name, user goes to visit detailed introduction of the team.

For each team, users can veiw its performance on different grand prix, information includes how many scores its racers earn and team score. When you click on history, page will be forwarded to wikipedia where users can get more information.

Racer page, which also displayed as default showing all racers is introduced earlier. After a visiter registers, he or she is allowed to view detailed information of a racer, personal information and performance in each match. Also, here I provide a link called biography to wikipedia for more information.

Grand prix page, you can also call it as circuit page. Each year F1 has 19 matches in 19 different circuits located in 19 different countries. In this website, grand prix tells users match time and winner for each match. I am still working on this page. When users click on name of a circuit, it is supposed to show a modal containing a picture of the circuit. Sounds great? It will come soon.

Match result

F1 audience is sure to know that each match is seperated to three sessions, free practice session, qualifying session and formal race. A car's starting postion on race is dependant on racer's perfomance in qualifying race. Thus, in this page, I design two tables to show race information. In qualifying result table, it displays results in decreasing order of racer's best time. Race result table tells us time each racer spent to finish competition and points they earned.

Fun forum module

Forum provides funs to give comments about the match and drivers. Before each match, administrator will encourage to predict who is gonna win the match and what kind of strategy a team is gonna apply. A shortcoming for this part is that, now it is only a public area which means every message is open to every user. So in the near future, I will add a function that a user can also send private messagess to another user he or she wants to communicate.Notice that at the right top of the page, there is a mail box that gives a remind when new message arrives.
